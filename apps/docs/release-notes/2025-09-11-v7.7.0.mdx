## What's new

### CSRF double-submit token protection

All authenticated API requests now use a CSRF double-submit cookie pattern. The client reads the token from a cookie and sends it in a header that the server validates, blocking cross-site request forgery attempts. Existing sessions automatically get a new CSRF cookie issued if they don't have one.

### Cleaner field mapping dropdown

In the load records field mapping, the field type used to render as a badge that crowded polymorphic lookups. It now shows as tertiary text next to the field name, matching how field types are displayed everywhere else in the app.

### Smaller UI fixes

- The virtualized combobox control now sizes correctly in layouts where it was previously miscalculating.
- The error popover's nubbin is now the right color so it blends with the popover body.
- The XML sanitizer has been fixed and the Chrome extension does a better job of detecting when you're on a Salesforce domain.
