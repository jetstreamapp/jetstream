## What's new

### Smoother single sign-on

This release focuses on making SSO sign-in feel more reliable:

- Logging in to the desktop app no longer asks you to click "login" a second time when you weren't
  already signed in to the web app.
- The browser extension's login flow was reworked to avoid race conditions that could trip up
  authentication when several requests happened at once.
- When SSO resolves to an ambiguous account — for example, more than one possible identity match — you
  now get a specific, actionable error message that guides you toward fixing it, rather than a generic
  failure.

### Other fixes

- Unknown URLs now serve the landing page directly while keeping the original address visible in access
  logs, so bookmarks and mistyped links behave more sensibly.
