## What's new

### Quick filters keep working when the input is messy

Quick filters let you narrow a table as you type, and many people lean on regular expressions to do it.
Previously, typing a pattern that wasn't yet valid — a stray bracket, an unfinished group — could throw
and leave the table in a bad state. Now the filter recognizes an invalid expression and keeps the table
usable, so you can keep typing and refining without it breaking under you.

### Field mapping is more forgiving during loads

When you map a file's columns to Salesforce fields, the mapping is built up as you go and can briefly be
incomplete. Loads now handle missing or undefined mapping entries gracefully instead of surfacing an
error partway through setup, which makes getting a load configured feel a lot less fragile.

### A round of authentication and session hardening

This release tightens up several parts of signing in and managing sessions:

- The OAuth login flow now generates, stores, and validates a `state` parameter, which protects against
  cross-site request forgery (CSRF) during login.
- Revoking a session clears **all** of that user's tokens, and error details are preserved when a session
  ends — so if something goes wrong mid-login, the reason isn't lost when you're redirected.
- Expired access tokens are now handled during decryption instead of failing outright.
- Sessions track the authentication provider and account id, giving Jetstream a more accurate picture of
  a user's identity across different login methods.

Most of this is invisible day to day — it's the kind of work that quietly makes logging in and staying
logged in more dependable.
