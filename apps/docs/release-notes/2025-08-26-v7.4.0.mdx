## What's new

### Per-org encryption for stored credentials

We moved to a stronger encryption strategy for Salesforce org credentials. Each stored org now has its own encryption key instead of sharing a single global key across the database. This narrows the impact if a key is ever compromised and brings the access token decryption path into tighter typing on the backend.

No action is required. Existing orgs continue to work.

### Windows desktop login fix

In Windows development builds, the deep link that completes OAuth login back to Jetstream desktop was not being handled correctly. Login now completes as expected when running the desktop app on Windows.
