## What's new

### Bulk report deletes report success correctly

A production bug caused bulk report deletes from query results to show as 100% failed — even though the
reports were actually being deleted. Internally, a successful Salesforce delete returns a `204 No Content`
response, and a code path was returning that as an error before the real response was handed back. The
result was a wall of "failed" messages for deletes that had in fact succeeded. That's now fixed across
web, desktop, the browser extension, and canvas, so bulk deletes report their true outcome.

### Dark-mode polish

Following the SLDS 2 refresh, this release fixes a number of dark-mode display issues where elements
weren't rendering correctly. As part of the cleanup, bespoke color styles were normalized to use standard
Salesforce style hooks, which keeps Jetstream visually consistent with Salesforce and makes future theme
work more reliable.

### SSO login-provider settings behave correctly

When SSO is enabled, it's now recognized as a valid login provider, so you can disable other providers
without hitting validation errors. Jetstream also prevents you from disabling SSO when it's the only login
provider selected, avoiding a configuration that would lock your team out.

### Protecting your team's last admin

Teams are now prevented from removing or deactivating their final admin, so you can't accidentally leave a
team with no one able to manage it.

### Other fixes

- Clearer domain-verification messaging that distinguishes a stale record from a genuine verification
  failure, and points you to the correct location for hosting your verification file.
- More informative error messages on failed API requests.
- Additional SSO/SAML security hardening, including stable assertion replay protection and redaction of
  sensitive information from error reports.
