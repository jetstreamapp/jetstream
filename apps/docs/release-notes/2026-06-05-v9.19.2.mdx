## What's new

### Correct validation for Long Text Area and HTML fields

Length validation for Long Text Area and rich-text (HTML) fields was off, which could reject valid sizes
or allow invalid ones. The validation messages and logic now correctly enforce a length between 256 and
131,072 characters, matching Salesforce's own limits.

### Server-side request hardening

This release tightens up how Jetstream makes outbound requests, guarding against server-side request
forgery (SSRF):

- Stricter validation of the network requests involved in SSO.
- Better URL validation for the manual request endpoints you can call from Jetstream.
- Unnecessary headers are stripped when proxying a request to Salesforce.
